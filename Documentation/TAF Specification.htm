TAF Specification

Introduction

TAF is a program which allows to perform test automation process. It contains from two modules:

  1. TAF Core - standalone application. Execute Unified Test Scenario, prepare Execution List for Automation Tool TAF and generates Test Results Report
  2. Automation Tool TAF - Project for some automation tool, which performs operations listed in Execution List and creates Test Results Summary.

TAF Core

Responsibilities

  • Read Unified Test Scenario (keyword-driven approach)
  • Read Bug List
  • Generate Execution List
  • Generate Test Result Report by info from Test Results Summary and Bug List.

Features

  • Test Suites support - possibility to run several scenarios at once with the same or different Automation Tool TAF.
  • Test Results Report is created in html and xml formats.
  • Display links to related bugs in Test Results Report.
  • Possibility to mark failed test cases as "known issue".
  • Constants such as server url could be specified in configuration file.
  • TestResultsDirectory parameter is specified in TAF.exe.config. All test results reports (html and xml) have to be created in the folder, specified in this parameter.
  • Load/Repeat keywords support which allows to create data-driven test cases.
  • Generate excel with manual test cases from unified test scenario.
  • Kill automation tool process with all child processes when specified timeout is elapsed.

Unified Test Scenario

Unified Test Scenario - testing scenario, which contains set of test cases in format which is readable by functional testers and Test Automation Framework.

Unitified Test Scenario is an excel file consisting of at least to sheets - Description and TestCases. Description sheet can contains information about test scenario, such as Name, Version, Author, Description and History. TestCases sheet consists from a set of lines. Each line have specific meaning, depending on what is specified in 'Keyword' column. 'Comments' column can contain any comments about specific lines or test cases. Empty line stops test scenario execution.

Columns

Keyword Name Data #1 Data #2 Data #3 Data #4 Data #5 Comment

Basic keywords

Application - specify application or application module where you are testing.

TestCase - separate different test cases. Name of test case is specified in 'Name' column, ID is in 'Data #1' column.

State - navigate to specific screen in application or set application to specific state. State name specified in 'Name' column, data columns are not used. 

Action - execute action step, correspond to regular test case step. Name of action is specified in 'Name' column, action parameters are specified in data columns.

Verify - verifies something in application, correspond to verification step from expected results. 'Name' of verify is specified in 'Name' column, parameters in data columns.

Additional keywords

Assign - assign value to variable in order to reuse this value in next steps by specifying variable in curve braces ({}). Name of variable is specified in 'Name' column, variable value is specified in 'Data #1' column. Assign keyword also allows to calculate values of expressions, specified in 'Data #1' column and assign final value to variable.

Unique - works as simple assign, but adds at the end of variable unique numbers, so every time test case executes variable value will be different. You can specify '#' simbols in order to generate numeric values by specific masks, e.g. Unique|a|### will assign random value from 100 to 999 to variable 'a'; Unique|b|##.## will assign random decimal value, e.g. 29.79. In order to generate unique letters, not digits, use '$' as a mask. You can combine '#','$' and other symbols in order to generate custom variables, e.g. Unique|s|$$$##-auto will generage something like 'dza401-auto'.

Load - loads variable value from data table. Data table could be specified in comma- and semi-comma-separated list or as a table on separate sheet of Unified Test Scenario. If using a table on separate sheet it can have column "Description" for each value except for the first. If using the "Repeat" keyword with parameter "For all table rows" or specified test data range "For 5-10 row" that description will be added to the "Description" column of TestResults.xml.

Repeat - repeats execution of test case or test group specified number of time or for all rows in data table, which was used in nearest Load keyword. The Repeat keyword with parameter eg. "For 8-12 rows" will to repeat Test Case 5 times with test data from 8 to 12 rows. Please, check 2.3.7 'Load/Repeat keywords usage' for more details.

TestGroup - joins logically similar test cases in one group. It allows visually separate different groups of test cases in the detailed log of Automation tool. Name of test group is specified in 'Name' column.

Excel features compatibility

Calculated cells value - Excel calculations can be used for any cell in any column in Unified Test Scenario. However, make sure that calculated cell is of General type, and any numbers used in formulas are stored in cells of Numeric or General type (Format cells->Category menu). Otherwise, calculated cells can be recognized by TAF improperly.

After changing cell type, force Excel to use new type for cell by re-typing cell value (clicking on cell value and pressing Enter will be enough) or using Excel Error Checking tool.

TAF Core Advanced Features Description

Manual test cases generation

In order to generate from unified test scenario test cases for manual passing, you have to specify 'ManualTestCasesDirectory' option in configuration file, e.g. <ManualTestCasesDirectory>Manual Test Cases</ManualTestCasesDirectory>. After that you have to run TAF.exe for required scenario and this configuration and together with ExecutionList in manual test cases directory will be created excel file called as unified test scenario with postfix '(manual)'.

In configuration file you can specified path to template for manual test cases file in 'ManualTestCasesTemplate' option, e.g. <ManualTestCasesTemplate>ManualTestCasesTemplate.xls</ManualTestCasesTemplate>. Template should be valid Excel file, placed in directory specified in 'ManualTestCasesDirectory'. File should have spreadsheet 'Test Cases' and first line of this spreadsheet should contain the following columns:

  • ID
  • Application
  • Screen
  • Description
  • Steps
  • Expected Results
  • Comments

Second line in excel file should contains cells with example of formatting all test cases which will be generated.

If 'ManualTestCasesTemplate' option is not specified, default built-in TAF Core template for manual test cases will be used.

Test Suite Timeout

Timeout for test suite could be specified. If test is executed, but test suite timeout is elapsed, this test is interrupted and for the last test case in html report in 'Details' column message 'Automation tool was interrupted because test suite timeout was elapsed (#)', where '#' - timeout value from test suite. Tests, which should be executed after interrupted one, should not be executed.

Variables Reuse

Same variable values can be loaded from "Variables.txt" in different tests by automation TAF tool if "Variables.txt" file is stayed untouched. To reset variables, start TAF with --DeleteVariables parameter. "Variables.txt" will be replaced with blank file at test start.

Exit Codes

For each test result there is appropriate exit code:

  • 0 - Passed
  • 1 - Failed
  • 2 - Error (exception in automation tool occured)
  • 3 - Not Executed (dependent test case failed)
