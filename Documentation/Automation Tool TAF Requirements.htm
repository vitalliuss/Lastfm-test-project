Automation Tool TAF Requirements

1. Minimal (without test scenario/execution list support)

  1. Have to run from command line and be closed silently.
  2. Have to generate Test Results Summary (TestResultsSummary.txt) in tab-separated text format. Test Results Summary example could be found in Examples folder of TAF Core package
  3. Test Results Summary should contains the following columns:
    1. Test Case ID - test case ID.
    2. Description - test case description.
    3. Status - test case status (Passed/Failed/Not Executed).
    4. Comment - comment from execution list + any additional.
    5. Details - any details found during test case execution, e.g. reason of test case failure.
    6. Execution Time - test case execution time.

Minimal functional scheme of Automation Tool TAF:

2. Basic

  1. Automation tool TAF have to execute Execution List (ExecutionList.txt). Execution List example could be found in Examples folder of TAF Core package. Execution List have to be read line by line and special functions should be run dependently of specified keywords in the first line.
    1. Step name in format "Application.Step".
    2. TestScenario - general top-level bookmark in log (optional keyword).
    3. TestGroup - bookmark for group of similar test cases (optional keyword)
    4. TestCase - determines beginning of next test case.
    5. Assign - assign value to global variable, that could be passed in any function call.

Basic functional scheme of Automation Tool TAF:

3. Additional

  1. DependsOn - parameter for TestCase keyword. Execution of test case with such parameter depends on specified test case execution status. For example:

    TestCase | TC2 | Test case #2 | DependsOn(TC1)

    This record means that TC2 should be executed if TC1 status is 'Passed' only.

  2. ExecutionCondition - parameter for TestCase keyword. Execution of such test case depends on result of specified boolean expression. For example:

    TestCase | TC1 | Test case #1
    Assign | row | =GetRowNumber(Grid=Users, login=Vasily)
    TestCase | TC2 | Test case #2 | ExecutionCondition({row}>0)
    ...

    This record means that TC2 should be executed if value of "row" is greater than 0.
